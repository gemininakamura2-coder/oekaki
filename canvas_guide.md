# 🎨 Canvas.jsx 完全解説ガイド（学習用）

このドキュメントは、**`client/src/components/Canvas.jsx`** がどのような仕組みで「リアルタイムなお絵描き」を実現しているかを、Reactの基本機能を中心に解説するガイドです。

`Lobby.jsx` が「文字の入力」を扱うファイルだったのに対し、`Canvas.jsx` は「マウスや指の動き（座標）」を扱う、このアプリの心臓部にあたるファイルです。

---

## 1. そもそも `Canvas.jsx` とは何をしているファイル？
HTMLには、元々絵を描くための `<canvas>`（キャンバス）という画用紙のようなタグが用意されています。
このファイルは、その画用紙をReactの中で使いやすくカスタマイズし、以下の役割を持たせています。

1. **線を引く処理**: マウスが押し込まれて動いた軌跡を、PCの画面上に線として描く。
2. **座標のデータ化（正規化）**: スマホでもPCでもズレないように、線の位置を「0〜1の割合」に変換する。
3. **他人の線を描く**: サーバーから送られてきた「友達が書いた線のデータ」を受け取り、自分の画用紙にも自動で線を引く。

---

## 2. コードの構造・学習ポイントの解説

### ① インポート（3つの強力な道具）
```javascript
import React, { useRef, useEffect, useState } from 'react';
```
ここではReactの「三大機能（Hooks）」を全て使っています。
* **`useState`**: データの変化を記憶する（例：今マウスのボタンが押されているか？ `isDrawing`）。
* **`useRef`**: 画面上の本物のHTML要素（`<canvas>`）を**直接つかんで操作する**ための「マジックハンド」です。
* **`useEffect`**: 特定のタイミング（「画面が表示された時」や「データが届いた時」）に**自動でプログラムを走らせる**ための「見張り番」です。

---

### ② useRef による「画用紙」と「筆」の準備
```javascript
  const canvasRef = useRef(null);
  const contextRef = useRef(null);
```
お絵描きをするには、「本物の画用紙」と「筆（コンテキスト）」に直接触る必要があります。Reactでは、HTMLを直接操作することをあまり良しとしないため、この **`useRef`** を使って安全に画用紙を掴みます。

一番下のHTML部分で `<canvas ref={canvasRef} />` と書くことで、`canvasRef.current` の中に本物の画用紙が格納されます。

---

### ③ useEffect (1): 初回マウント時（最初の1回だけ実行）
```javascript
  useEffect(() => {
    const canvas = canvasRef.current;
    if (!canvas) return;
    
    // ...キャンバスの大きさや解像度を設定する処理...

    const context = canvas.getContext('2d');
    context.lineCap = 'round'; // 線の端を丸くする
    contextRef.current = context; // 「筆」も保存しておく
  }, []); // 👈 この空の配列 [] が「最初の1回だけやってね」という合図です
```
このブロックは、「画用紙が画面にパッと表示された最初の瞬間」にだけ走り、画用紙の大きさを決めたり、筆の毛先を丸く整えたりする準備体操をしています。

---

### ④ useEffect (2): 外部からのデータ監視
```javascript
  useEffect(() => {
    if (!externalStroke) return;
    drawFromData(externalStroke); // 届いたデータを使って線を引く
  }, [externalStroke]); 
  // 👈 配列の中に externalStroke を入れると、「このデータが変化するたびに実行してね」という合図になります。
```
親コンポーネント（`App.jsx`）から「ほら！友達が引いた線のデータ（`externalStroke`）がトランシーバーから届いたよ！」と新しいデータをもらうたびに、この `useEffect` が自動で反応し、友達の線を自分の画用紙に描きます。

---

### ⑤ マウス操作と「描画イベント」の流れ
ユーザーが絵を描くときは、3つのステップで関数が連続して呼ばれます。

1. **`startDrawing` (描き始め)** : `<canvas onMouseDown={...}>`
   * マウスがポチッと押された瞬間。
   * 現在の座標を記憶し、`isDrawing` を true (描いている状態) にする。
2. **`draw` (描いている最中)** : `<canvas onMouseMove={...}>`
   * ボタンを押したままマウスを動かすたびに、**1秒間に数十回**もの猛スピードで呼ばれ続ける関数。
   * 「さっき記憶した座標」から「今マウスがある座標」まで、短い直線を引く。
   * そしてすぐに**「今引いた線のデータ」を `onStrokeEmit` を使って親に渡し**、サーバーへ送信してもらう。
3. **`stopDrawing` (描き終わり)** : `<canvas onMouseUp={...}>`
   * マウスのボタンを離した瞬間。`isDrawing` を false に戻す。

---

### ⑥ 魔法の座標変換（割合への変換）
```javascript
    // 画面上で線を引く (ローカル描画)
    ctx.moveTo(lastPos.x, lastPos.y);
    ctx.lineTo(x, y);
    ctx.stroke();

    // 友達にデータを送る用 (onStrokeEmit)
    onStrokeEmit({
      fromX: lastPos.x / rect.width,  // 例: 100px / 500px = 0.2 (20%)
      fromY: lastPos.y / rect.height,
      toX: x / rect.width,
      toY: y / rect.height,
      ...
    });
```
ここが `Canvas.jsx` における最大の数学の工夫です。
自分の画面に線を引く時はそのままのピクセル（100pxなど）を使いますが、**サーバー（友達）に送る時は、Canvas全体の横幅・縦幅で割り算をして「0.0 〜 1.0 の割合（パーセント）」に変換（正規化）しています。**

そして、友達からデータをもらって描く関数（`drawFromData`）の中では、この逆（`データ * 自分のCanvasの幅`）の掛け算をすることで、元の位置に完璧に復元しています。

---

## 🌟 学習のポイントまとめ
`Canvas.jsx` で使われている一番大事なReactの概念は以下の2つです。

1. **`useRef`** : 画面上の要素（キャンバス）を直接つかんだり、文字通り「画面に表示させないけど覚えておきたいデータ（コンテキスト）」を持っておきたい時に使う。
2. **`useEffect`の依存配列 (`[ ]` の中身)** : 空っぽ `[]` なら最初だけ。変数 `[data]` を入れたら「その変数が変わるたびに」関数を自動で動かしてくれる**賢い自動プログラム機**。

この「キャンバス」と「React」の融合処理は、ゲームを作ったりアニメーションを作ったりするときに100%登場する黄金パターンです！
